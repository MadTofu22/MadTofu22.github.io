# 1 - What is the command for listing the files in the current folder (directory)?
  'ls'

# 2 - What is the command for changing directories?
  'cd' or 'chdir'

# 3 - What is the command for going back a folder (to a parent directory)?
  'cd ..'

# 4 - What is a quick way to create a new folder?
  'mkdir [directory name]'
  ex. mkdir test

# 5 - What output would you expect when you run the command 'ls' in an empty folder? What about a folder containing files?
  In an empty folder 'ls' does not return an output.
  In a folder containing files 'ls' will return a list of each file and folder in the current directory.

# 6 - What is a quick way to create a new file?
  'touch [File Name]'
  ex. touch new_file.txt

# 7 - Run 'git --version' in your terminal. What is the output?
  'git --version' will return the current installed version of Github.
  My output is 'git version 2.24.3 (Apple Git-128)'

# 8 - Why would you run the command 'git status'?
  To check if there are any un-staged, un-tracked, and/or un-committed changes to the repository.
  Or to check to make sure you staged, tracked, and/or committed changes correctly.

# 9 - What does the command 'git add .' do?
  'git add .' stages all files in the current git directory.

# 10 - What is the command to push your committed changes to your repo on GitHub?
  'git push origin master' will push the committed changes to the master branch.
  Other branches may be pushed to, but we have not created any separate branches for this assignment.
